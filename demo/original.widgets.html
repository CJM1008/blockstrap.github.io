blockstrap widgets

self-hosted widgets included within the framework

 

DONATION BUTTONS

Donate

Add the bs-donate class to any element to activate widget

Balance: 0.00000000 | Donations: 0

The only required options here are address and chain - which should be added to the HTML data-attributes as follows:

<a 
                
    href="#" 
    class="bs-donate" 
    data-address="naj1C...vGS54" 
    data-chain="doget"
    
>Donate</a>
Other HTML Data-Attribute Options:

  • data-qr default is set to true

    Displaying QR Codes requires the qrcode.js dependency.

  • data-bip default is set to true

    Would you like to display Desktop QT Link?

  • data-amount default is set to 0

    Include defined amount in BIP Link and Modal Header?

  • data-inject default is set to true

    Attempt to auto-inject modal HTML?

  • data-group default is set to false

    If updating text we use this as the class and add the dynamic text to an element with the following class, as follows (and seen above):

    <span class="dontate-txs-naj1CrvX2kBXa7uDJMRfiUAnWD5ZFvGS54">0</span>

  • data-balance default is set to false

    Update balances based on group class and address.

  • data-txs default is set to false

    Update TX counts based on group class and address.

  • data-loading default is set to false

    This is the text to display prior to fetching dynamic content and will use the content within the HTML if nothing specific has been set.

  • data-label default is set to false

    This allows you to add a label to Desktop QT links.

GENERATION BUTTONS

Generate Keys

Add bs-generate class to any element to activate widget

(change the demo parameters below)

The only required options here are salt and chain - which should be added to the data-attributes as follows:

WARNING
Failure to use unique salts could leave these keys and any coins they control vulnerable to brute force 
<a 

    href="#" 
    class="bs-generate" 
    data-salt="your-unique-app-salt" 
    data-chain="doget"

>Generate Keys</a>
Other HTML Data-Attribute Options:

  • data-random default is set to true

    Allow random salt generation

  • data-deterministic default is set to true

    Allow deterministic salt generation

  • data-callback default is set to true

    This allows you to control what happens once the keys have been generated. By default it calls the bs_default_generate_callback function, which creates a modal window containing the key pair.

  • data-print default is set to true

    If using the default callback, show the print keys button?

  • data-save default is set to true

    If using the default callback, show the save keys button?

ACCESS SAVED ACCOUNTS

View Accounts

Add bs-accounts-modal class to element to activate


This widget does not have any required options, but can utilize the following if set to HTML data attributes:

  • data-all default is set to false

    If set to true it will show all Blockstrap accounts rather than just accounts created by widgets.

PAYMENT BUTTONS

Payment Request

Add the bs-request class to any element to activate widget

(change the demo parameters below)

Required options here include currency, chains, amount, addresses and salt - all of which should be added to the HTML data-attributes as follows:


<a 

    href="#" 
    class="bs-request" 
    data-salt="your-unique-app-salt" 
    data-currency="sgd"
    data-chains="btc, ltc, dash, doget"
    data-amount="1"
    data-addresses="1G..Mt, Lb..hP, etc"

>Request Payment</a>

We provide support for 165 currencies, where the corresponding amount is converted and presented via a choice of chains. Deterministic payment addresses are created whilst waiting for specific transactions that are then routed to the corresponding addresses. Please note that the chains and addresses values should be comma separated arrays that follow the same order.

Other HTML Data-Attribute Options:

  • data-amounts default is set to 0

    If currency is set to false or is unsupported this comma separated array can be used in the same order as the chains to provide crypto-only payment requests.

    If set to 0 and required due to invalid currency, twice the mining fee of the chain will be requested.

  • data-callback default is set to true

    This allows you to control what happens once the payment has been routed correctly. By default it calls the bs_default_payment_callback function, which generates a modal with information about the transaction.

  • data-qr default is set to true

    Displaying QR Codes requires the qrcode.js dependency.

  • data-bip default is set to true

    Would you like to display Desktop QT Link?

  • data-poll default is set to 60

    How many seconds between polls?

  • data-title default is set to Payment Request

    The title of the modal window

  • data-description default is set to false

    The description given to the payment request, which if set to false will use the text within the button as the description.
